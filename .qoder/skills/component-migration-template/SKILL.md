# 通用组件迁移技能

## 概述

本技能提供一套**通用的组件迁移方法论**，基于对 Icon、Form、Search 等组件实际迁移经验的提炼总结，适用于组件库中任何组件的版本升级与结构改造场景。

## 迁移分析维度

当面对一个组件的迁移任务时，从以下 **7 个维度** 进行系统性分析：

### 1. 文件结构变化

分析组件文件组织方式的演进：

| 模式            | 描述                                                                   | 适用场景                         |
| --------------- | ---------------------------------------------------------------------- | -------------------------------- |
| 单文件 → 模块化 | 从单一 `.vue` 拆分为 `type.ts`/`composable.ts`/`style.ts`/`[name].vue` | 逻辑复杂度上升，需要分离关注点   |
| 平铺 → 目录化   | 从同级目录移至独立子目录                                               | 组件逻辑、类型、样式需要独立管理 |
| 合并 → 拆分     | 多个功能合并的组件拆分为子组件                                         | 关注点分离，提升可维护性         |
| 内联 → 抽取     | 内联 Props/逻辑抽取为独立文件                                          | 类型复用，逻辑共享               |

**方法：** 追踪文件路径和结构变化，对比各版本的文件列表和目录结构

### 2. 组件 API 变更

分析 Props、Emits、Expose 等外部接口的变化：

- **Props：** 新增/废弃/重命名的属性，默认值变化，类型变化
- **Emits：** 新增事件，事件签名变化，事件命名规范变化
- **Expose：** 暴露的方法变化，返回值变化
- **Slots：** 插槽名称/作用域变化

**方法：** 对比各版本 `defineProps` / `defineEmits` / `defineExpose` 定义

### 3. 依赖关系变化

分析组件内外依赖的调整：

- **外部依赖 → 自包含：** 将外部模块依赖移入组件自身（如 `IconConfig` 从 `@/config` 移至 `composable.ts`）
- **工具函数替换：** 替换底层工具库（如 `shareMemoryCache` → `pendingRequests`）
- **依赖移除：** 去除不再需要的导入引用
- **新增依赖：** 引入新的工具函数或库

**方法：** 对比各版本的 `import` 语句变化

### 4. 核心逻辑演进

分析组件核心功能逻辑的改进：

- **算法优化：** 缓存策略、请求去重、性能优化
- **错误处理增强：** 增加校验、降级策略、错误日志
- **异步模式变化：** 回调 → Promise → async/await
- **响应式机制：** watch 深度/范围变化、computed 添加/移除

**方法：** 对比核心函数签名、执行流程、错误处理路径

### 5. 编码规范演进

分析代码风格和规范的提升：

- **TypeScript 类型强化：** 内联类型 → 接口/类型别名 → 泛型约束
- **命名规范：** 组件名、CSS 类名、事件名、key 前缀的规范化
- **注释完善：** JSDoc 文档的添加，`@param`/`@return`/`@throws` 标注
- **导出策略：** 最小暴露原则，公共类型统一导出

**方法：** 检查类型定义、命名风格、注释覆盖率和导出列表

### 6. 默认值调整

分析组件默认行为的变化：

- **Props 默认值：** 数值/字符串/布尔值的变化（如 `size: '32'` → `size: 24`）
- **行为默认值：** 组件默认启用/禁用某项功能
- **样式默认值：** 默认尺寸、颜色、间距的调整

**方法：** 对比 `withDefaults` 和各版本配置默认值

### 7. 测试与文档

分析测试和文档的配套变化：

- **测试覆盖：** 新增/修改/删除的测试用例
- **文档更新：** API 文档、示例代码、迁移指南
- **类型导出：** d.ts 声明文件的变化
- **Storybook/演示：** 演示页面的更新

## 迁移工作流

### Phase 1: 差异分析

```
Task:
1. [ ] 收集组件的所有可用版本（来自 VCS 历史、备份文件或新旧两份源码）
2. [ ] 对每个版本，分析以下信息：
     - 文件路径和结构
     - Props/Emits/Expose API
     - 核心函数签名
     - import/export 依赖
     - 配置/默认值
3. [ ] 使用 diff 工具（如 Beyond Compare、diff、IDE 内置对比功能）逐文件对比各版本差异
4. [ ] 将所有变更整理为上述 7 个维度的对比表格
```

### Phase 2: 变更评估

评估每次变更的影响范围：

| 变更类型     | 破坏性 | 兼容性影响          |
| ------------ | ------ | ------------------- |
| 文件结构变化 | 是     | 导入路径需更新      |
| Props 重命名 | 是     | 调用方需适配        |
| Props 新增   | 否     | 可选兼容            |
| 默认值变化   | 潜在   | 视觉/行为可能不一致 |
| 依赖变化     | 潜在   | 需验证构建无错误    |
| 命名变化     | 是     | 组件引用名需更新    |
| 逻辑优化     | 否     | 功能等价/增强       |

### Phase 3: 迁移执行

按以下优先级顺序执行迁移：

1. **创建目录结构** — 建立目标文件结构，迁移类型定义
2. **提取公共逻辑** — 抽取 composable/style 等独立模块
3. **迁移核心视图** — 改造 `.vue` 文件主体
4. **更新导入导出** — 修正 `index.ts` 和外部引用
5. **适配 API 变化** — 处理 Props/Emits 变更
6. **调整默认值** — 更新 `withDefaults`
7. **更新测试** — 适配测试用例
8. **更新文档** — 更新 API 文档和示例

### Phase 4: 验证确认

```
验证项清单（根据组件实际情况勾选）：

文件结构:
- [ ] 新目录结构已建立，旧文件已删除
- [ ] 模块入口 index.ts 正确导出所有公共类型和组件

API:
- [ ] Props 定义完整，默认值正确
- [ ] Emits 事件签名正确
- [ ] Expose 最小暴露原则

依赖:
- [ ] 所有 import 路径正确
- [ ] 旧的外部依赖引用已移除
- [ ] 循环依赖检测通过

功能:
- [ ] 组件渲染正常，无控制台错误
- [ ] Props 响应式更新正常
- [ ] 事件触发/冒泡正常
- [ ] 边界情况（空数据、错误状态）处理正常

构建:
- [ ] TypeScript 编译无错误
- [ ] 构建产物正确
- [ ] 类型声明文件正确导出
```

## 实际案例参考

本技能基于以下组件的实际迁移经验提炼：

- **Icon 组件**（单文件 `Icon.vue` → `Icon/` 目录，含 `type.ts`/`composable.ts`/`index.ts`/`Icon.vue`）
  - 组件名：`Icon` → `QIcon`
  - 缓存机制：`shareMemoryCache` → `pendingRequests` 并发去重
  - 默认尺寸：`'32'` → `24`
  - 配置方式：外部 `@/config` → 自包含
