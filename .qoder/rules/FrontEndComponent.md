## 前端组件开发规范

在进行前端组件开发时，必须遵循标准的目录结构，并清晰、完整地定义组件的属性、事件、外部 API 以及外部使用类型，以确保组件的高内聚、低耦合及良好的可维护性。

### 1. 标准单组件目录结构

- **基础文件组成**：标准单组件应放置在 `src/` 目录下，并由以下文件组成，以实现逻辑、样式与视图的分离：
  - `type.ts`：定义组件相关的所有类型。
  - `composable.ts`：封装组件的组合式函数与核心逻辑。
  - `style.ts`：处理组件的样式相关逻辑。
  - `[componentName].vue`：组件的模板与视图层。
- **子组件管理**：如果组件包含私有子组件，需在当前组件目录下新增 `component` 文件夹进行集中管理。

### 2. 属性定义 (Props)

- **类型安全**：必须使用 TypeScript 为所有 Props 定义明确的类型。
- **默认值与必填**：明确区分必填项与可选项，为可选项提供合理的默认值。
- **注释说明**：为复杂的 Props 添加 JSDoc 注释，清晰描述其功能、用途、可选值及影响。

### 3. 事件定义 (Events / Emits)

- **类型声明**：使用 `defineEmits` 并配合 TypeScript 泛型或类型声明，确保事件参数的类型安全。
- **命名规范**：事件命名应语义化，状态更新类事件推荐使用 `update:propName` 格式以支持 `v-model`。
- **注释说明**：为事件添加注释，说明触发时机及参数含义。

### 4. 外部 API 暴露 (Expose)

- **最小暴露原则**：仅通过 `defineExpose` 暴露外部确实需要调用的方法或属性，严格隐藏内部实现细节。
- **类型定义**：为暴露的 API 定义清晰的接口类型，方便外部获取组件实例后进行准确的类型推导。
- **注释说明**：为暴露的方法和属性添加完整的文档注释，说明调用方式及返回值。

### 5. 外部使用类型 (External Types)

- **统一导出**：将组件的 Props、Emits、Slots 及 Expose 类型统一在组件文件或独立的类型文件中导出，供外部直接引用。
- **辅助类型**：提供组件所需的枚举、联合类型或配置接口，提升外部使用时的开发体验（DX）。
- **导入导出规范**：保持导入导出干净清晰，仅导出必须暴露给外部使用的公共类型，隐藏内部辅助类型。
